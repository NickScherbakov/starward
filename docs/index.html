Get Started in 60 Seconds

Choose your preferred installation method

bash 
# Pull and run (starts in <60s)
docker pull ghcr.io/nickscherbakov/starward:latest
docker run -p 4566:4566 starward

# Create a snapshot
curl -X POST http://localhost:4566/snapshots

# List snapshots
curl http://localhost:4566/snapshots
bash 
# Install
pip install starward

# Start server
starward up

# In another terminal
starward status          # Check health
starward snapshot        # Create snapshot
starward parity          # View API coverage
starward cost            # See profiling report

# Stop server
starward down
bash 
# Clone
git clone https://github.com/NickScherbakov/starward.git
cd starward

# Install in dev mode
pip install -e ".[dev]"

# Run tests
pytest -v

# Start server
starward up

See It in Action

Real-world examples demonstrating Starward's capabilities

01

Terraform Migration

Test infrastructure changes locally without cloud costs. Create snapshots before risky changes and rollback instantly.

starward up -d
cd examples/terraform
terraform apply  # Provisions S3 bucket locally

starward snapshot --id before-migration
terraform apply  # Make changes
starward restore before-migration  # Rollback
Benefit: Iterate on Terraform without cloud API latency or costs.
02

Chaos Engineering

Inject latency and failures to test timeout handling and retry logic. See real-time metrics and SLA impact.

python examples/chaos_latency_test.py

# Output:
[CHAOS] Injecting 23.4ms latency into s3.create_bucket
Operation  1:  24.12 ms
[CHAOS] Injecting 47.8ms latency into s3.create_bucket
Operation  2:  48.56 ms
Benefit: Test resilience without touching production.
03

IAM Training

Interactive scenarios with auto-grading. Learn cloud security in a safe, guided environment.

starward learn run iam-basics

# Interactive walkthrough:
# 1. Create IAM role
# 2. Attach S3 read policy
# 3. Test permissions
# 4. Get instant feedback
Benefit: Team onboarding to cloud IAM in minutes, not weeks.
